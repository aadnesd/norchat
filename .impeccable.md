# Impeccable — Design Context

This file captures the non-inferrable design context for the Norway-first AI Support Agent Platform. All design work must start here. Do not re-derive this from the codebase.

Last confirmed: 2026-04-19.

---

## Design Context

### Users

The product has **two surfaces on one site**, and the design must serve both without visually splitting into "two products":

1. **Evaluators** — Nordic support-ops managers, heads of CX, and operations leads at SMB / scale-up companies (ecommerce, SaaS, customer operations). They arrive cold from search or a referral, skim during a workday, and decide in minutes whether to take the onboarding flow. They care about: Norwegian/EU data residency, GDPR defaults, time-to-value, and whether this is "serious" software.

2. **Operators** — The same people (plus their support leads and admins) after signing up. They spend real minutes inside the onboarding flow, then return to the agent settings, source ingestion, observability dashboard, and conversation review. They want density, keyboard-like efficiency, and an interface that respects that they know what they're doing.

**Context of use**: daytime, desk, office or WFH, mid-to-large monitors dominant, brief mobile checks secondary. Not a late-night tool, not a consumer app.

**Job to be done**: "Get an AI support agent from zero to deployed on our site in an afternoon, and then run it confidently — prove deflection, spot problems, fix them."

### Brand Personality

Three words: **sharp, operational, serious.**

- **Sharp** — precise typography, no decorative fluff, every element earns its place. Corners are tight (2–6px), rules are 1px, spacing is deliberate.
- **Operational** — this is a tool, not a storefront. Information density is a feature. Numbers, states, and timestamps are first-class. Copy is direct, never cute.
- **Serious** — adults at work, doing important work (customer trust is at stake). Quiet confidence. No emojis, no exclamation points, no growth-marketing patter. Compliance and clarity signal trust more than warmth does.

Emotional goals: **confidence, competence, calm focus.** Not delight. Not warmth. Not awe. The user should feel like they're using an instrument that respects their time.

### Aesthetic Direction

**Reference space**: Linear, Height, Vercel dashboard, Plain.com, Readwise admin — **app-operator minimalism** with editorial discipline. The marketing surface should feel like documentation for a serious product, not like a landing-page template.

**Explicit anti-references**:
- The current site's look (Fraunces + teal + warm radial gradient) — too wellness-SaaS, too soft.
- Generic Chatbase / Intercom Fin / Drift visual language (purple gradients, rounded 20px+ cards with drop shadows, cyan accents, hero screenshot on a gradient mesh).
- "Creative agency" maximalism — giant display type, asymmetric chaos, editorial photography treatments. Wrong register for operators.
- Developer-terminal pastiche (monospace everywhere, green-on-black, "$ npm install" decorative elements).

**Theme**: **Light only.** Daytime desk use, extended reading of dashboards, note-taking. A warm paper-white background (not pure white), not a gray tech background. The dashboard must be legible in office lighting.

**Color system**:
- **Neutrals**: warm off-white paper (`oklch(98.5% 0.004 95)` range), tinted toward the brand's accent hue. Text ink is a near-black with a faint warm cast, not `#000`.
- **Accent**: a single **muted forest / moss green** — `oklch(38% 0.06 150)` range as primary ink-accent, `oklch(52% 0.09 150)` for interactive. Grounded, Nordic landscape, NOT "tech teal" and NOT seafoam pastel. Used sparingly per the 60-30-10 rule — it appears on primary actions, key metrics, and nowhere else.
- **Status colors**: desaturated, paper-era — a dusty terracotta for errors, a muted amber for warnings, a soft olive for success. No neon, no saturated alerts.

**Typography**:
- **Display/Editorial face**: a serif with character — **GT Sectra**, **Tiempos Headline**, **PP Editorial New**, or **Söhne Breit** — not Fraunces, not Playfair, not Cormorant. Used for H1/H2 and the occasional pull quote. Tight tracking, modest sizes (this is not a magazine).
- **Body + UI face**: a neo-grotesk with operator clarity — **Söhne**, **ABC Diatype**, or if restricted to free fonts, **Geist** or **Work Sans** with careful weight choices — not Inter, not DM Sans, not Manrope. Numeric glyphs must be tabular for dashboards.
- **Numeric/data**: tabular lining figures from the body font (not a separate mono family). Monospace is reserved for actual code blocks (embed snippets).

**Spatial**:
- 4pt scale with semantic tokens (`--space-xs` … `--space-2xl`).
- Tight card radii (4–6px), 1px hairline borders over drop shadows. Shadows only on genuinely floating elements (popovers, the widget preview).
- Marketing surfaces get generous vertical rhythm; app surfaces get dense grids with purposeful whitespace between groups, not around every element.
- Left-aligned, not centered. Asymmetric information grids where appropriate.

**The unforgettable detail**: the product feels like a **workbench**, not a showroom. Numbers, timestamps, statuses, and state transitions are foregrounded with the same care a magazine gives to typography. A user should look at the dashboard and think "this was built by someone who has actually done support operations."

### Design Principles

1. **Density is respect.** Operators don't want cards floating in space. Put information where they can see it, grouped with intention, separated by hairlines and rhythm rather than padding balloons.

2. **One accent, used once.** The moss green signals "this is the primary action / this is the key metric." If everything is green, nothing is. Use ink black for most affordances; reserve the accent.

3. **Typography carries hierarchy, not color or size alone.** A 1.333 modular scale, two weights, and the display serif for moments that deserve authorship. Avoid the `h1` → `h6` step-ladder; use 3–4 sizes maximum.

4. **Hairlines over shadows; edges over gradients.** 1px borders in a tinted neutral define structure. Shadows are physical — they appear only when something is genuinely elevated.

5. **Numbers are first-class citizens.** Tabular figures, right-aligned in tables, always with units or context, never orphaned. The deflection-rate number is the single most important number on the dashboard; treat it that way.

6. **Copy is operational.** "Add website" not "Let's add your website 🎉". "Retrain" not "Refresh your knowledge". Error messages name the problem and the fix.

7. **No AI-slop tells.** No purple-blue gradients, no gradient text, no side-stripe accent borders on cards/alerts, no glassmorphism, no rounded icon tiles above every heading, no monospace-as-decoration.

## Implementation Notes (for future craft sessions)

- The current `apps/web/src/styles.css` predates this context and uses the old Fraunces/teal system. When executing `/impeccable craft ...`, treat it as legacy — introduce a new token layer in OKLCH and migrate surfaces as they're touched, don't leave the two systems visually fighting.
- Tailwind config is currently empty (`extend: {}`). A proper design token layer (CSS custom properties consumed by Tailwind via `theme.extend.colors`, `fontFamily`, `spacing`) is the right place to land this system.
- Playwright visual snapshots exist — expect them to need updating after the redesign, and flag that explicitly when it happens.
